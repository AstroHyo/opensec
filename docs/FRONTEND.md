# FRONTEND.md

OpenSec currently has no standalone web frontend.

The primary user interface is:

- Telegram digest text
- Telegram follow-up commands
- optional shell execution on the VPS

That means current "frontend" quality is mostly about:

- mobile readability
- compact information density
- safe Markdown escaping
- short, high-signal Korean copy with English product and repo names preserved

If a future dashboard is added, it should remain secondary to the Telegram-first workflow.
