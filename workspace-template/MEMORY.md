# MEMORY.md

Stable, durable facts and preferences should live here.

Suggested sections:

- communication preferences
- channel defaults and escalation norms
- important repos
- server inventory
- deployment conventions
- recurring automations
- do-not-forget constraints

Do not use this as a chronological work log.
Put raw notes in `memory/YYYY-MM-DD.md` and distill only stable facts here.

Do not put secrets here.
